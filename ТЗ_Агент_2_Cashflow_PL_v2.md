## 1. Загальний опис

Агент активується на уроці 2.2. До цього моменту в базі вже є `process_model` з блоку 1 — агент використовує його щоб не питати про бізнес з нуля.

Ключова відмінність від v1.0: агент збирає статті **одночасно для Cashflow і P&L**. Для кожної витрати він ставить одне просте уточнювальне питання яке визначає `cost_type` — і саме це поле визначає куди стаття потрапляє в P&L і який рівень прибутку вона формує.

Клієнт не чує слів «собівартість», «COGS», «OPEX» — він просто відповідає на зрозуміле питання. Агент сам класифікує.

### Мета агента

- Зібрати повний персоналізований список статей доходів і витрат
- Класифікувати кожну витрату для правильного відображення в P&L
- Визначити частоту кожної статті для Cashflow
- Зберегти JSON який одночасно живить обидва шаблони

### Що на виході

Єдиний JSON який автоматично розгортається в:
- **Google Sheets Cashflow** — хронологія руху грошей по статтях
- **Google Sheets P&L** — ієрархічний звіт з рівнями прибутку

## 3. Як статті використовуються в Cashflow і P&L

Cashflow і P&L використовують **одні і ті самі статті** — але по-різному їх відображають.

### Cashflow — плоска хронологія

```
НАДХОДЖЕННЯ
  + Консалтинг
  + Ретейнер
ВИТРАТИ
  - Зарплата Оксани
  - Підрядники
  - Реклама
  - Єдиний податок
= ЗАЛИШОК НА КІНЕЦЬ МІСЯЦЯ
```

### P&L — ієрархія рівнів прибутку

```
ДОХОДИ
  + Консалтинг
  + Ретейнер
= ВАЛОВИЙ ДОХІД

  — Собівартість (прямі витрати на виконання)
      Підрядники на проєкти
      Матеріали і ліцензії
= ВАЛОВИЙ ПРИБУТОК          ← скільки заробила послуга / продукт

  — Операційні витрати (загальні витрати бізнесу)
      Зарплати команди
      Реклама і маркетинг
      Оренда і комунальні
      Підписки на ПЗ
= ОПЕРАЦІЙНИЙ ПРИБУТОК      ← скільки заробив бізнес як система

  — Виплата власнику
= ПРИБУТОК ДО ПОДАТКІВ

  — Податки
= ЧИСТИЙ ПРИБУТОК           ← скільки реально залишилось
```

### Класифікація через одне просте питання

Кожна витрата отримує `cost_type` через одне питання агента:

> «[Назва] — вона виникає тільки коли є конкретне замовлення, чи є щомісяця незалежно від кількості клієнтів?»

| Відповідь | `cost_type` | Рівень P&L |
|---|---|---|
| «тільки під замовлення» | `cogs` | Собівартість → валовий прибуток |
| «є завжди» | `opex` | Операційні витрати → операційний прибуток |
| Виплата власнику | `owner` | Прибуток до податків |
| Податки | `tax` | Чистий прибуток |

## 4. Структура діалогу

### Блок А — Джерела доходів

```
А1: «Ось базові джерела доходу для [тип бізнесу]:
[список з бібліотеки]
Що з цього є у тебе? Що прибрати або додати?»

А2: «Ти продаєш кілька різних послуг — чи відрізняється їх ціна суттєво?
Якщо так — краще зробити окрему статтю для кожної.»

А3: «Є клієнти які платять тобі щомісяця фіксовану суму —
підписка, ретейнер, абонплата?»

А4: «Є якісь інші надходження крім основних продажів?»
```

→ *Всі доходи: `cost_type: "income"`, `pl_level: "revenue"`*

---

### Блок Б — Прямі витрати на виконання

Агент пояснює без терміну «собівартість»:

```
Б0: «Тепер поговоримо про витрати.
Спочатку — ті що виникають тільки тоді коли є конкретне замовлення.
Наприклад, підрядник під проєкт — пряма витрата.
Зарплата менеджера щомісяця — це вже інше.»

Б1: «Є витрати які виникають тільки під конкретне замовлення?
Для [тип бізнесу] зазвичай це:
[список cogs з бібліотеки]
Що з цього є?»

Б2 (уточнення):
«[Назва] — вона є тільки коли є замовлення,
чи платиш її щомісяця незалежно?»
```

→ *`cost_type: "cogs"`, `pl_level: "gross_profit"`*

---

### Блок В — Витрати на команду

```
В1: «У тебе в команді [N] людей: [список ролей].
Витрати на команду зазвичай:
• Зарплати штатних
• Оплата постійних підрядників
• Бонуси та премії
• ЄСВ
Що з цього є?»

В2: «Підрядники — постійні щомісяця чи під різні задачі?»
→ постійні → cost_type: "opex"
→ під проєкти → cost_type: "cogs"

В3: «Ти сам отримуєш гроші від бізнесу?»
→ cost_type: "owner", pl_level: "pre_tax_profit"
```

---

### Блок Г — Операційні витрати

```
Г1: «Ось витрати які є щомісяця незалежно від клієнтів:

📍 Приміщення: оренда, комунальні
📣 Маркетинг: реклама, SMM, сайт
💻 ПЗ: CRM, бухгалтерія, підписки
📦 Інше: зв'язок, банк, канцелярія

Що з цього є? Що додати?»

Г2: «Реклама — фіксований бюджет чи змінний?»
→ frequency: "monthly" або "irregular", cost_type завжди "opex"
```

→ *`cost_type: "opex"`, `pl_level: "operating_profit"`*

---

### Блок Д — Податки і фінансові витрати

```
Д1: «Як оформлений бізнес — ФОП, ТОВ, інше?»

Д2 (ФОП): «Яка група єдиного податку?
Єдиний податок + ЄСВ — фіксовані суми щокварталу.»

Д2 (ТОВ): «Податок на прибуток — щомісяця чи щокварталу? ПДВ є?»

Д3: «Є кредити або позики?»

Д4: «Банк бере комісію? Еквайринг є?»
```

→ *`cost_type: "tax"`, `pl_level: "net_profit"`*

---

### Фінальний підсумок

```
Ось як буде виглядати твій P&L:

💰 ДОХОДИ: [список]
— Прямі витрати: [cogs]
= ВАЛОВИЙ ПРИБУТОК

— Витрати бізнесу: [opex]
= ОПЕРАЦІЙНИЙ ПРИБУТОК

— Виплата тобі: [owner]
= ПРИБУТОК ДО ПОДАТКІВ

— Податки: [taxes]
= ЧИСТИЙ ПРИБУТОК

Нічого не пропустили?
```

Після підтвердження → `###CASHFLOW_ITEMS_COMPLETE###`

### Довідник `cost_type`

| Значення | Рівень P&L | Пояснення |
|---|---|---|
| `income` | Доходи | Всі надходження |
| `cogs` | Собівартість | Витрати тільки під конкретне замовлення |
| `opex` | Операційні витрати | Постійні витрати бізнесу |
| `owner` | Виплата власнику | Особиста винагорода власника |
| `tax` | Податки | Всі податкові платежі |

### Довідник `pl_level`

| Значення | Що показує |
|---|---|
| `revenue` | Рядок доходів |
| `gross_profit` | Впливає на валовий прибуток |
| `operating_profit` | Впливає на операційний прибуток |
| `pre_tax_profit` | Впливає на прибуток до податків |
| `net_profit` | Впливає на чистий прибуток |

### Довідник `frequency`

| Значення | Опис |
|---|---|
| `monthly` | Щомісяця |
| `quarterly` | Щокварталу |
| `annual` | Раз на рік |
| `project_based` | Під конкретний проєкт |
| `irregular` | Нерегулярно |

## 8. Логіка бота (псевдокод)

```javascript
async function handleCashflowAgent(user_id, text) {

  const session = await db.getSession(user_id)

  if (!session.process_model || session.process_model.status !== 'complete') {
    return sendMessage(user_id, 'Спочатку потрібно завершити урок 1.2.')
  }

  if (!session.cashflow_session) {
    session.cashflow_session = {
      history: [],
      items: { income: [], cogs: [], team: [], operations: [], taxes: [] },
      pl_structure: {},
      completed_blocks: [],
      status: 'in_progress'
    }
  }

  session.cashflow_session.history.push({ role: 'user', content: text })

  const library = getItemsLibrary(session.process_model.business_type)

  const response = await callLLM({
    system: CASHFLOW_AGENT_PROMPT
      .replace('{{business_type}}', session.process_model.business_type)
      .replace('{{team_size}}', session.process_model.team_size)
      .replace('{{team_roles}}', extractRoles(session.process_model))
      .replace('{{items_library}}', JSON.stringify(library))
      .replace('{{completed_blocks}}', session.cashflow_session.completed_blocks.join(', '))
      .replace('{{collected_items}}', JSON.stringify(session.cashflow_session.items)),
    messages: session.cashflow_session.history
  })

  const updatedItems = extractJSON(response)
  if (updatedItems) {
    session.cashflow_session.items = updatedItems
    // Автоматично будуємо pl_structure з cost_type кожної статті
    session.cashflow_session.pl_structure = buildPLStructure(updatedItems)
  }

  if (response.includes('###CASHFLOW_ITEMS_COMPLETE###')) {

    session.cashflow_session.status = 'complete'
    await db.saveSession(session)

    // Генеруємо обидва шаблони одночасно
    const [cashflowUrl, plUrl] = await Promise.all([
      createCashflowTemplate(session.cashflow_session.items),
      createPLTemplate(
        session.cashflow_session.items,
        session.cashflow_session.pl_structure
      )
    ])

    await sendMessage(user_id, COMPLETION_MESSAGE)
    await sendMessage(user_id, `Cashflow-шаблон: ${cashflowUrl}`)
    await sendMessage(user_id, `P&L-шаблон: ${plUrl}`)
    return
  }

  const bot_text = extractText(response)
  session.cashflow_session.history.push({ role: 'assistant', content: bot_text })
  await db.saveSession(session)
  await sendMessage(user_id, bot_text)
}

// Автоматично будує pl_structure з cost_type кожної статті
function buildPLStructure(items) {
  const all = [
    ...items.income, ...items.cogs,
    ...items.team, ...items.operations, ...items.taxes
  ]
  return {
    revenue:          all.filter(i => i.cost_type === 'income').map(i => i.id),
    cogs:             all.filter(i => i.cost_type === 'cogs').map(i => i.id),
    gross_profit:     'revenue - cogs',
    opex:             all.filter(i => i.cost_type === 'opex').map(i => i.id),
    operating_profit: 'gross_profit - opex',
    owner_payout:     all.filter(i => i.cost_type === 'owner').map(i => i.id),
    pre_tax_profit:   'operating_profit - owner_payout',
    taxes:            all.filter(i => i.cost_type === 'tax').map(i => i.id),
    net_profit:       'pre_tax_profit - taxes'
  }
}
```
