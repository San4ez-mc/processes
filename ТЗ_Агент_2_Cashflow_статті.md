## 1. Загальний опис

Агент активується на уроці 2.2 після того як учасник переглянув вступне відео про структуру Cashflow. До цього моменту в базі вже є `process_model` з блоку 1 — агент використовує його щоб не питати про бізнес з нуля.

Ключова відмінність від звичайного опитування: агент **не просто питає «які у вас витрати»**, а одразу пропонує готовий список під тип бізнесу і просить підтвердити або скоригувати. Це знімає тривогу «а раптом щось забуду» і скорочує час діалогу вдвічі.

### Мета агента

- Зібрати повний персоналізований список статей доходів і витрат
- Не пропустити типові статті для даного типу бізнесу
- Визначити частоту і характер кожної статті (регулярна / нерегулярна)
- Зберегти результат у JSON для автоматичного створення Cashflow-шаблону

### Що на виході

Готовий список статей який автоматично розгортається в Google Sheets шаблон Cashflow з правильними категоріями, підкатегоріями і структурою.

## 2. Що агент знає на старті

Агент отримує `process_model` з блоку 1. З нього він бере:

| Поле | Звідки | Навіщо |
|---|---|---|
| `business_type` | process_model | Визначає базовий набір статей |
| `team_size` | process_model | Розуміє масштаб витрат на команду |
| `lanes[].role` | process_model | Знає які ролі є — пропонує витрати під них |
| `lanes[].responsible` | process_model | Знає імена — персоналізує повідомлення |

**Важливо:** агент починає НЕ з нуля. Він каже:

> «Я вже знаю що у тебе [тип бізнесу] з командою [N] людей. На основі цього я підготував базовий список статей — зараз будемо його уточнювати разом.»

## 3. Структура діалогу

Діалог складається з 4 блоків. Кожен блок — окрема тема. В кожному блоці агент спочатку **пропонує готовий список**, потім уточнює деталі.

---

### Блок А — Джерела доходів

**Мета:** зрозуміти звідки приходять гроші і скільки окремих статей доходу потрібно.

**Логіка:**
1. Агент пропонує базову структуру доходів під тип бізнесу
2. Питає чи є різниця між продуктами/послугами по ціні
3. Питає чи є регулярні платежі (підписки, ретейнери, абонплата)
4. Питає чи є інші джерела доходу (партнерські, реферальні тощо)

**Питання блоку А:**

```
А1: «Ось базові джерела доходу для [тип бізнесу]:
[список з бібліотеки]

Що з цього є у тебе? Що прибрати або додати?»

А2 (якщо кілька послуг/продуктів):
«Ти продаєш кілька різних послуг — чи відрізняється їх ціна суттєво?
Якщо так — краще зробити окрему статтю для кожної.
Назви свої основні послуги або продукти.»

А3: «Є клієнти які платять тобі щомісяця фіксовану суму —
підписка, ретейнер, абонплата?»

А4: «Є якісь інші надходження грошей крім основних продажів?
Наприклад, партнерські виплати, відсотки, оренда чогось?»
```

→ *Результат: масив `income_items` в JSON*

---

### Блок Б — Витрати на команду

**Мета:** зібрати всі витрати пов'язані з людьми.

**Логіка:** агент знає ролі з process_model — пропонує статті під кожну роль.

**Питання блоку Б:**

```
Б1: «У тебе в команді [N] людей з такими ролями: [список ролей].
Зазвичай витрати на команду включають:
• Зарплати штатних співробітників
• Оплата підрядників і фрілансерів
• Бонуси та премії
• ЄСВ (якщо є офіційно оформлені)

Що з цього є у тебе?»

Б2 (якщо є підрядники):
«Підрядники у тебе — це постійні люди які працюють щомісяця,
чи різні люди під різні задачі?»
→ якщо постійні: окрема стаття на кожного або загальна «Підрядники»
→ якщо різні: загальна стаття «Підрядники»

Б3: «Ти сам отримуєш зарплату або дивіденди від бізнесу?
Як ти це оформляєш зараз?»
```

→ *Результат: масив `team_expense_items` в JSON*

---

### Блок В — Операційні витрати

**Мета:** зібрати всі регулярні витрати на роботу бізнесу.

**Логіка:** агент пропонує список по категоріях і просить підтвердити кожну.

**Питання блоку В:**

```
В1: «Ось типові операційні витрати для [тип бізнесу].
Відзнач що є у тебе:

📍 Приміщення
• Оренда офісу / коворкінгу
• Комунальні послуги

📣 Маркетинг
• Реклама (Facebook, Google, TikTok)
• SMM і контент
• Сайт і домен

💻 Програмне забезпечення
• CRM система
• Бухгалтерська програма
• Інші підписки (Slack, Notion, Zoom тощо)

📦 Інше
• Мобільний зв'язок
• Банківське обслуговування і комісії
• Канцелярія і офісні витрати

Що з цього є? Що додати?»

В2 (уточнення по рекламі якщо є):
«Реклама у тебе — це фіксований бюджет щомісяця
чи змінюється залежно від сезону?»

В3 (уточнення по оренді якщо є):
«Офіс — ти орендуєш постійно чи використовуєш коворкінг
або працюєш віддалено?»
```

→ *Результат: масив `operational_expense_items` в JSON*

---

### Блок Г — Специфічні витрати

**Мета:** зібрати витрати специфічні для типу бізнесу — собівартість, виробництво, логістика тощо.

**Логіка:** цей блок адаптується під `business_type` з process_model.

```
Для послуг:
Г1: «Є витрати безпосередньо пов'язані з наданням послуги?
Наприклад, матеріали, витратники, інструменти, ліцензії?»

Для продукту / торгівлі:
Г1: «Є закупівля товару або сировини?
Як часто — щомісяця чи нерегулярно?»
Г2: «Є витрати на логістику і доставку?
Хто платить за доставку — ти чи клієнт?»
Г3: «Є склад або витрати на зберігання?»

Для виробництва:
Г1: «Є закупівля сировини і матеріалів?»
Г2: «Є витрати на обладнання або його обслуговування?»
Г3: «Є витрати на електроенергію і комунальні для виробництва?»
```

→ *Результат: масив `specific_expense_items` в JSON*

---

### Блок Д — Податки і фінансові витрати

**Мета:** не пропустити податки і банківські витрати.

```
Д1: «Як у тебе оформлений бізнес — ФОП, ТОВ, щось інше?»

Д2 (для ФОП):
«Ти на єдиному податку? Яка група?
Єдиний податок + ЄСВ — це фіксовані суми щокварталу.
Додаємо їх як окремі статті.»

Д2 (для ТОВ):
«Податок на прибуток і ПДВ — вони у тебе є?
Як часто платиш — щомісяця чи щокварталу?»

Д3: «Є кредити або позики у бізнесі?
Виплати по них теж варто зафіксувати окремою статтею.»

Д4: «Банк бере комісію за обслуговування рахунку або за перекази?
Еквайринг — беруть відсоток від безготівкових оплат?»
```

→ *Результат: масив `tax_and_finance_items` в JSON*

## 6. Системний промпт

```
Ти — ШІ-асистент курсу "Фінансова система малого бізнесу".
Твоя задача — допомогти власнику бізнесу скласти персоналізований
список статей доходів і витрат для Cashflow.

## ЩО ТИ ЗНАЄШ ПРО БІЗНЕС
Тип бізнесу: {{business_type}}
Розмір команди: {{team_size}} людей
Ролі в команді: {{team_roles}}

## ПРАВИЛА ДІАЛОГУ
- Задавай ОДНЕ питання за раз.
- Завжди пропонуй готові варіанти — не змушуй вигадувати з нуля.
- Формат пропозиції: список з bullet points + питання «що з цього є?»
- Після підтвердження — коротко підсумуй і переходь далі.
- Тон: дружній, як фінансовий консультант який добре знає цей тип бізнесу.
- Не використовуй терміни: "стаття", "категорія", "JSON". Говори просто.

## СТРУКТУРА ОПИТУВАННЯ
Проведи клієнта через 5 блоків по черзі:
А — Джерела доходів
Б — Витрати на команду
В — Операційні витрати
Г — Специфічні витрати (під тип бізнесу)
Д — Податки і фінансові витрати
```

## 7. Логіка бота (псевдокод)

```javascript
async function handleCashflowAgent(user_id, text) {

  const session = await db.getSession(user_id)

  // Перевіряємо що блок 1 завершений
  if (!session.process_model || session.process_model.status !== 'complete') {
    return sendMessage(user_id, 'Спочатку потрібно завершити урок 1.2 — побудову бізнес-процесу.')
  }

  // Ініціалізуємо cashflow сесію якщо перший раз
  if (!session.cashflow_session) {
    session.cashflow_session = {
      history: [],
      items: { income: [], cogs: [], team: [], operations: [], taxes: [] },
      completed_blocks: [],
      status: 'in_progress'
    }
  }

  session.cashflow_session.history.push({ role: 'user', content: text })

  // Готуємо бібліотеку статей під тип бізнесу
  const library = getItemsLibrary(session.process_model.business_type)

  // Запит до LLM
  const response = await callLLM({
    system: CASHFLOW_AGENT_PROMPT
      .replace('{{business_type}}', session.process_model.business_type)
      .replace('{{team_size}}', session.process_model.team_size)
      .replace('{{team_roles}}', extractRoles(session.process_model))
      .replace('{{items_library}}', JSON.stringify(library))
      .replace('{{completed_blocks}}', session.cashflow_session.completed_blocks.join(', '))
      .replace('{{collected_items}}', JSON.stringify(session.cashflow_session.items)),
    messages: session.cashflow_session.history
  })

  // Оновлюємо зібрані статті
  const updatedItems = extractJSON(response)
  if (updatedItems) {
    session.cashflow_session.items = updatedItems
  }

  // Перевіряємо завершення
  if (response.includes('###CASHFLOW_ITEMS_COMPLETE###')) {

    session.cashflow_session.status = 'complete'
    await db.saveSession(session)

    // Генеруємо Google Sheets шаблон
    const sheetsUrl = await createCashflowTemplate(
      session.cashflow_session.items,
      session.process_model.business_type
    )

    await sendMessage(user_id, COMPLETION_MESSAGE)
    await sendMessage(user_id, `Ось твій персоналізований Cashflow-шаблон:\n${sheetsUrl}`)
    return
  }

  // Звичайна відповідь
  const bot_text = extractText(response)
  session.cashflow_session.history.push({ role: 'assistant', content: bot_text })
  await db.saveSession(session)
  await sendMessage(user_id, bot_text)
}
```
